## 🔄 주요 변경 및 이슈 포인트 (2023 → 2025)

### 1. **Next.js (13 → 16)**
- **App Router 안정화**: 13.x에서 도입된 App Router가 16.x에서 기본 권장 구조로 자리잡음.  
  - 기존 `pages/` 기반 프로젝트는 마이그레이션 필요.  
- **React Server Components (RSC)**: Flight 프로토콜 관련 보안 취약점 패치됨.  
  - 구버전에서 최신 버전으로 올리면 RSC 동작 방식이 일부 변경되어 호환성 문제 발생 가능.  
- **Middleware 개선**: Edge Runtime과의 통합 강화 → 기존 미들웨어 코드 수정 필요.  
- **빌드 시스템**: Turbopack 도입으로 빌드 속도 개선, Webpack 기반 설정 일부 비호환.  

---

### 2. **React (18 → 19)**
- **Concurrent Features 확장**: React 18에서 도입된 Concurrent Rendering이 19에서 더 강화됨.  
  - 일부 라이브러리(특히 오래된 상태 관리 라이브러리) 호환성 문제 발생 가능.  
- **Server Components**: Next.js와 함께 사용 시 API 변경 반영 필요.  
- **Strict Mode 강화**: 개발 환경에서 더 많은 경고 발생 → 레거시 코드 수정 필요.  

---

### 3. **Node.js (18/20 → 22/25)**
- **보안 패치**: OpenSSL, V8 엔진 업데이트로 구버전 대비 보안성 강화.  
- **ESM(ECMAScript Modules) 기본화**: CommonJS 기반 코드 일부 수정 필요.  
- **Native Fetch API**: Node.js 18에서 실험적이던 Fetch가 22에서 안정화 → 기존 `axios`/`node-fetch` 의존성 줄일 수 있음.  
- **패키지 호환성**: 오래된 npm 패키지들이 Node.js 22/25에서 동작하지 않을 수 있음.  

---

## ⚠️ 예상되는 마이그레이션 이슈
- **라우팅 구조 변경**: `pages/` → `app/` 전환 시 대규모 코드 리팩토링 필요.  
- **보안 취약점 패치 반영**: RSC Flight 관련 CVE-2025-55182, CVE-2025-66478 등.  
- **빌드/배포 환경 차이**: Turbopack 도입으로 CI/CD 설정 변경 필요.  
- **라이브러리 호환성**: React 19와 Node.js 22에서 일부 오래된 라이브러리 동작 불가.  
- **테스트 코드 수정**: Strict Mode 강화로 인해 테스트 실패 가능성.
